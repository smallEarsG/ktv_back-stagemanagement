# 超级管理系统 - 页面设计（Desktop-first）

## 0. 全局设计规范

### Layout
- 框架：桌面优先（>=1200px），主体采用居中定宽容器（如 1200–1440px）+ 两侧留白。
- 栅格：内容区使用 CSS Grid（表单/筛选区 2–4 列自适应），列表区使用表格（Table）。
- 间距：8px 基准（8/16/24/32）。主要区块使用 24px padding。
- 响应式：
  - >=1200px：全量展示筛选项与表格列
  - 768–1199px：筛选项自动换行，表格列可水平滚动
  - <768px：仅基础可用（可折叠筛选 + 列表卡片化为后续优化点，不作为本期必需）

### Meta Information（默认）
- title：超级管理系统
- description：超级管理员登录并进行商家管理的后台系统
- Open Graph：
  - og:title：超级管理系统
  - og:description：商家管理后台

### Global Styles（Design Tokens）
- 背景色：#F5F7FA（页面底色）
- 内容卡片：#FFFFFF，圆角 8px，阴影 0 2px 8px rgba(0,0,0,0.06)
- 主色：#1677FF（主按钮/链接）
- 危险色：#F5222D（停用/删除风险操作）
- 文字：
  - 主标题 20px/28px，次标题 16px/24px，正文 14px/22px
  - 颜色：#1F2329（主文案），#667085（次文案）
- 按钮：
  - Primary：主色填充；hover 加深；disabled 降低透明度
  - Secondary：白底描边；hover 背景浅灰
- 链接：主色；hover 下划线
- 表单校验：错误提示红色文本 + 输入框红色描边

### 全局交互与状态
- Loading：列表区骨架屏/加载条；按钮提交态禁用重复提交
- Empty：空状态插画/文案 + “清空筛选/刷新”操作
- Error：顶部提示条/Toast + 可重试按钮
- 鉴权失效：统一弹 Toast（“登录已失效，请重新登录”）并跳转登录页

## 1. 登录页（/login）

### Meta
- title：登录 - 超级管理系统
- description：使用管理员账号登录系统

### Page Structure
- 单页居中布局：
  - 页面背景（浅灰）
  - 中央卡片（登录框）

### Sections & Components
1) 顶部品牌区
- Logo（可选）+ 文案“超级管理系统”

2) 登录卡片（Card）
- 标题：登录
- 表单（纵向布局）
  - 账号输入框（Input）
  - 密码输入框（Password Input，支持显隐）
  - 登录按钮（Primary Button）
- 校验与错误展示
  - 必填校验：账号/密码为空时提示
  - 接口错误：展示后端返回的错误信息（统一格式化）
- 提交行为
  - 点击登录 -> 调用登录接口 -> 成功保存鉴权信息 -> 跳转 `/merchants`

3) 页脚（轻量）
- 版权/组织信息（可选）

### Interaction States
- 登录按钮：loading + 禁用
- 登录成功：Toast（可选）+ 跳转
- 登录失败：表单下方/顶部提示条展示错误

## 2. 商家管理页（/merchants）

### Meta
- title：商家管理 - 超级管理系统
- description：查询与维护商家信息

### Page Structure
- 典型后台布局（上-下）：
  - 顶部导航栏（Header）
  - 内容区（Container）
    - 查询筛选卡片
    - 列表卡片（表格 + 分页）

### Sections & Components
1) 顶部导航栏（Header）
- 左侧：系统名“超级管理系统”
- 右侧：当前用户标识（如账号名，若接口提供）+ 退出登录按钮
- 退出登录：清理本地鉴权信息并跳转 `/login`

2) 查询与过滤区（Filter Card）
- 关键词输入框（placeholder 示例：输入商家名称/编号）
- 查询按钮（Primary）
- 重置按钮（Secondary）
- （可选占位）后续扩展筛选项区域：仅在接口明确支持时再增加

3) 操作区（List Toolbar）
- 新增商家按钮（Primary）
- 刷新按钮（Secondary/图标按钮）

4) 商家列表（Table Card）
- 表格列（以接口返回为准，前端不假设更多字段）
  - 商家名称/编号/状态/更新时间（示例，实际按接口调整）
  - 操作列：编辑、启用/停用
- 行内操作
  - 编辑：打开“商家表单抽屉/弹窗”并回填数据
  - 启用/停用：二次确认弹窗（明确风险文案）
- 状态展示
  - Loading：表格加载态
  - Empty：无结果提示 + “重置筛选”
  - Error：加载失败提示 + “重试”

5) 分页（Pagination）
- 页码切换；页大小切换（若接口支持）
- 切换分页时保留当前筛选条件

6) 新增/编辑商家（Drawer / Modal）
- 标题：新增商家 / 编辑商家
- 表单字段：严格以接口要求为准（本期不在设计稿中虚构字段）
- 按钮：取消 / 保存
- 保存逻辑
  - 提交中禁用
  - 成功：关闭弹窗 + 刷新列表 + 成功提示
  - 失败：保留弹窗并展示后端错误

### 鉴权与请求头（页面层约束体现）
- 进入页面前：路由守卫检查是否存在有效登录态；无则跳转 `/login`
- 页面内所有数据请求：必须自动携带 `X-Store-Id: 0`；鉴权头按接口约定注入
- 若请求返回鉴权失效：立即跳转登录页并提示
